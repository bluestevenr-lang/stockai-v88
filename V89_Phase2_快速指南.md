## 📈 性能面板指标说明

| 指标 | 含义 | 目标值 |
|------|------|--------|
| 数据获取耗时 | 从API获取数据的时间 | < 5000ms |
| 计算耗时 | 指标计算和分析的时间 | < 1000ms |
| 渲染耗时 | UI渲染的时间 | < 1000ms |
| 总耗时 | 从开始到结束的总时间 | < 3000ms |
| 缓存命中率 | 命中次数/总请求次数 | > 80% |
| 缓存项数量 | 当前缓存的数据项数 | 3-10项 |
| 过期兜底 | 使用过期缓存的次数 | 0次（理想） |
| 错误次数 | 数据获取失败的次数 | 0次（理想） |

## 🎯 性能优化建议

### 场景1：首次访问慢
**原因**：冷启动需要获取数据  
**优化**：无需优化，后续会自动加速

### 场景2：热启动仍慢
**检查**：
1. 查看缓存命中率（应 > 80%）
2. 检查网络连接
3. 查看错误次数

### 场景3：缓存命中率低
**原因**：TTL设置过短  
**优化**：调大 `CACHE_TTL_*` 参数

### 场景4：数据更新不及时
**原因**：TTL设置过长  
**优化**：
1. 调小 `CACHE_TTL_*` 参数
2. 或点击"强制刷新"按钮

## 🐛 常见问题

### Q1: 性能面板不显示
**原因**：开关关闭  
**解决**：确认 `Config.ENABLE_PERF_LAYER = True`

### Q2: 强制刷新无效
**检查**：
1. 观察终端日志
2. 确认缓存已清除
3. 检查是否有错误提示

### Q3: 缓存命中率始终0%
**原因**：每次都强制刷新  
**解决**：不要频繁点击"强制刷新"按钮

### Q4: 性能评级显示"较慢"
**原因**：
1. 网络慢
2. 数据量大
3. 计算复杂

**优化**：
1. 检查网络
2. 调大缓存TTL
3. 减少并发数（降低CPU负载）
