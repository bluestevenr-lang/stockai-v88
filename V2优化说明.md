### 2. AI 成本优化（省钱核心）✅

#### 优化项1：固定输出上限
```python
generation_config={
    'max_output_tokens': 800,  # 限制输出长度（省钱）
    'temperature': 0.7,         # 降低创造性，提高准确性
}
```
- **效果：** 每次报告 600-1200 中文字，Token 消耗减少 50%

#### 优化项2：精简上下文
**优化前：**
```python
# 冗余的详细数据
data_summary = f"- {name}({symbol}): {price:.2f} ({change:+.2f}%), 趋势: {trend}, MA5: {ma5:.2f}..."
```

**优化后：**
```python
# 只传核心涨跌
data_summary = f"{name} {price:.1f} 📈{abs(change):.1f}%"
```
- **效果：** 上下文 Token 减少 70%

#### 优化项3：减少重试次数
**优化前：** 最多 3 次重试  
**优化后：** 最多 2 次重试，指数退避（1s / 3s）
- **效果：** 失败场景下 Token 消耗减少 33%

#### 优化项4：结果缓存
```python
# 同一时段（早报/晚报）不重复生成
cache_key = f"{date}_{period}_{report_type}"
CACHE_TTL = 3600  # 缓存1小时
```
- **效果：** 测试/调试时 100% 避免重复调用

#### 优化项5：模板化输出
**优化前：** AI 生成完整报告（800-1000 字）  
**优化后：** 固定模板，AI 只填核心观点（600-800 字）

```python
# 模板结构固定，AI只输出：
# - 市场解读（一句话）
# - 提前量预测（具体数值）
# - 操作建议（仓位+板块）
```
- **效果：** 输出 Token 减少 40%，质量更聚焦

### 3. 报告内容优化 ✅

#### 强调"提前量"和"预测"
**早报重点：**
- 今日开盘预测（高/平/低开，具体幅度）
- 全天走向（区间+关键点位）
- 最可能活跃的 2 个板块

**晚报重点：**
- 明日开盘预判（具体 %，理由）
- 全天走势区间+关键点位
- 明天最可能爆发的 2 个板块
- 隔夜外盘传导路径

#### 输出格式优化
- 去掉废话寒暄
- 必须给出具体数值（点位/百分比）
- 突出"预判"和"埋伏"机会
- 严格控制字数
