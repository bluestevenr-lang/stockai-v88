5. AI将分析：
   - 📰 重大新闻事件
   - 💬 市场整体情绪（评分0-100）

#### 📈 个股舆情分析

1. 在侧边栏搜索股票（例如：`AAPL`）
2. 进入 **⚔️ 深度作战室**
3. 找到 **🏦 机构研究报告** 部分
4. 切换到 **📰 舆情分析** Tab（第4个Tab）
5. 点击 **🚀 启动舆情分析** 按钮
6. AI将生成完整的舆情报告：
   - 📰 近期重要新闻动态（3-5条）
   - 📊 新闻解读与影响分析（利好/利空/中性）
   - 💬 市场情绪评估（评分0-100）
   - 🎯 舆情对股价的影响预判（短期+中期）
   - 🎖️ 舆情综合评级（投资建议）
   - ⚠️ 风险提示

## 🔄 与其他模块的协同

### 1. 与宏观预期层协同
- 舆情分析时会自动考虑市场体制（Risk On/Off）
- 在Risk Off环境下，舆情评分会更保守
- 提示词中包含市场环境背景

### 2. 与机构研究协同
- 舆情分析作为机构研究报告的第4个Tab
- 与"个股深度研究"、"机会雷达"、"风险预警"并列
- 形成完整的投资决策链：技术面 + 基本面 + 舆情面

### 3. 与持仓管理协同
- 持仓个股可以快速跳转到深度分析
- 深度分析包含舆情分析
- 可评估持仓股票的舆情风险

## ⚠️ 注意事项

### 1. API Key 必需
- 舆情分析需要 Gemini API Key
- 如未配置，会显示提示信息
- 配置方法：在代码中设置 `MY_GEMINI_KEY` 或使用 Streamlit Secrets

### 2. 网络要求
- 舆情分析需要调用Gemini AI API
- 请确保代理配置正确（port: 1082）
- 单次分析耗时约10-20秒

### 3. 缓存机制
- 已分析的舆情结果会缓存在 `st.session_state`
- 刷新页面会清空缓存
- 可重复点击按钮更新分析

### 4. AI生成内容
- 舆情分析由AI生成，仅供参考
- 不构成投资建议
- 请结合技术面、基本面综合判断

## 🚀 性能优化

### 1. 按需触发
- 所有舆情分析都是按钮触发
- 不会自动运行，节省API调用

### 2. 进度反馈
- 4步进度条（0% → 25% → 50% → 75% → 100%）
- 每步显示当前任务
- 完成后自动隐藏

### 3. 结果缓存
- 使用 `st.session_state` 缓存
- 避免重复分析同一个股/市场
- 手动点击可强制刷新
