# 📱 钉钉对话功能说明

## ✅ 已实现功能

### 1. 持仓股管理（通过钉钉对话）

在钉钉群中发送指令，机器人会自动更新 `my_portfolio.xlsx`：

#### 📌 支持的指令格式：

```
# 1️⃣ 增加股票（简单模式）
持仓股：增加 AAPL

# 2️⃣ 增加股票（完整模式）
持仓股：增加 AAPL 苹果 150.5 100
格式：持仓股：增加 [代码] [名称] [成本] [数量]

# 3️⃣ 删除股票
持仓股：删除 AAPL

# 4️⃣ 查询当前持仓
持仓股：查询
或直接发：持仓股
```

#### ✅ 自动响应：

- ✅ 添加成功 → 回复确认消息
- ✅ 删除成功 → 回复确认消息
- ✅ 查询持仓 → 返回当前所有持仓列表
- ❌ 错误操作 → 返回错误提示

## 🚀 部署方式

### 方式1：本地测试（需要内网穿透）

#### 步骤1：安装依赖
```bash
pip install flask pandas openpyxl
```

#### 步骤2：配置环境变量
```bash
export DINGTALK_WEBHOOK="你的钉钉Webhook"
export DINGTALK_SECRET="你的钉钉Secret"
```

#### 步骤3：启动服务
```bash
python dingtalk_bot.py
```

#### 步骤4：配置钉钉机器人
1. 打开钉钉群 → 群设置 → 智能群助手 → 添加机器人
2. 选择"自定义机器人" → 添加
3. 安全设置：选择"加签"，复制Secret
4. 获取Webhook地址
5. **重要**：添加"Outgoing机器人"
   - 在机器人设置中，启用"消息接收"
   - 填写你的服务器地址：`http://你的公网IP:5000/webhook`
   - 触发词：`持仓股`

#### 步骤5：使用内网穿透（测试用）
```bash
# 使用 ngrok
ngrok http 5000

# 或使用 localtunnel
npx localtunnel --port 5000
```

将生成的公网地址填入钉钉机器人的"消息接收地址"。

### 方式2：云服务器部署（推荐）

#### 适用场景：
- 需要24小时在线
- 稳定接收钉钉消息

#### 推荐平台：
1. **阿里云 ECS** - 轻量应用服务器
2. **腾讯云 CVM** - 云服务器
3. **AWS EC2** - t2.micro（免费套餐）

#### 部署步骤：
```bash
# 1. 上传代码到服务器
scp dingtalk_bot.py user@server:/home/user/

# 2. 安装依赖
pip install flask pandas openpyxl

# 3. 使用 systemd 设置开机自启
sudo nano /etc/systemd/system/dingtalk-bot.service

# 4. 添加配置：
[Unit]
Description=DingTalk Bot Service
After=network.target

[Service]
User=your_user
WorkingDirectory=/home/user/StockAI
Environment="DINGTALK_WEBHOOK=你的webhook"
Environment="DINGTALK_SECRET=你的secret"
ExecStart=/usr/bin/python3 dingtalk_bot.py
Restart=always

[Install]
WantedBy=multi-user.target

# 5. 启动服务
sudo systemctl start dingtalk-bot
sudo systemctl enable dingtalk-bot
```
